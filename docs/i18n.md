# Internationalization (i18n)

The app is multilingual: **English is the source language**, with **German, French
and Italian** translations. Users pick a language from the navbar switcher (home
screen only); the choice is stored in a cookie *and* persisted to
`account.Profile.language` so it follows the account across devices.

## Why a custom toolchain?

Django's standard `makemessages` / `compilemessages` require the GNU **gettext**
binaries (`xgettext`, `msgfmt`), which are **not installed** on the dev machine.
So this project uses a small self-contained replacement:

| Piece | Role |
|-------|------|
| `locale/source_messages.py` | **Source of truth.** Maps each English msgid ŌåÆ `(de, fr, it)`. Two dicts: `DJANGO` (templates + Python) and `DJANGOJS` (JavaScript). |
| `scripts/manage_translations.py` | Extracts msgids from code, diffs them against the table (`--check`), and writes `locale/**/*.po` + `*.mo` from the table (`--build`). |
| `locale/<lang>/LC_MESSAGES/*.po` + `*.mo` | **Generated** ŌĆö do not hand-edit; `--build` overwrites them. Django reads the `.mo` at runtime. |

If you ever install gettext, you can switch back to the standard Django commands;
the `.po` files are standard format.

## How it's wired

- **Settings** (`CQCPathfinder/settings.py`): `LANGUAGE_CODE="en"`, `LANGUAGES`
  (en/de/fr/it), `LOCALE_PATHS=[BASE_DIR/"locale"]`, `LocaleMiddleware` (after
  `SessionMiddleware`), and the `i18n` template context processor.
- **URLs** (`CQCPathfinder/urls.py`): `set_language` (`/i18n/setlang/`) and the JS
  catalog `JavaScriptCatalog` (`/jsi18n/`), both `login_not_required` (they must
  work on the public login page despite the global `LoginRequiredMiddleware`).
- **Switcher + catalog** (`templates/base.html`): the nav language dropdown, the
  dynamic `<html lang>`, and `<script src="{% url 'javascript-catalog' %}">` loaded
  before `{% block scripts %}` so `gettext()` is global to all page JS.
- **Persistence**: `account.views.set_language` saves the choice to the profile;
  `CQCPathfinder.views.LocaleLoginView` re-applies `Profile.language` on login.

## Adding / changing translatable text ŌĆö the workflow

1. **Mark the string in code with its English text:**

   | Layer | How |
   |-------|-----|
   | Template | `{% load i18n %}` once at top, then `{% trans "Save" %}` |
   | Template attr | `title="{% trans 'Close' %}"`, `placeholder="{% trans 'SearchŌĆ”' %}"` |
   | Python | `from django.utils.translation import gettext_lazy as _` ŌåÆ `_("Save")` |
   | JavaScript | `gettext("Save")` ŌĆö the global is provided by `/jsi18n/` |

   Conventions:
   - **Plurals**: avoid Django plural blocks (the extractor doesn't parse them).
     Use two msgids + a conditional: `{% if n == 1 %}{% trans "topic" %}{% else %}{% trans "topics" %}{% endif %}`, or in JS `n === 1 ? gettext('topic') : gettext('topics')`.
   - **Variables in a string**: keep the variable outside the call, e.g.
     `` `${gettext('Control')} ${n}` ``, or JS `interpolate(gettext('ŌĆ” %s ŌĆ”'), [x])`.
   - **Never translate a value used as a key/identifier.** If a string is both a
     label and a lookup key (e.g. the results `STATUS` map, the stats chart `who`),
     keep a stable English/raw `key` for logic and translate only the display label.
   - **Error messages** (backend failures, `alert(...)`, `Fehler: ŌĆ”`) are
     intentionally left untranslated per product decision ŌĆö don't wrap them.

2. **Add the translations** to `locale/source_messages.py` in the right domain dict:

   ```python
   "Save": ("Speichern", "Enregistrer", "Salva"),
   ```

3. **Check and build:**

   ```bash
   python scripts/manage_translations.py --check    # lists msgids in code missing from the table
   python scripts/manage_translations.py --build     # writes .po + .mo
   ```

   `--check` exits non-zero if any code msgid lacks a table row (also reports
   unused rows). Treat a clean `--check` as the gate.

4. **If you changed JavaScript or CSS**, run collectstatic (manifest-hashed static
   via servestatic) and restart the preview, or the change won't show:

   ```bash
   python manage.py collectstatic --noinput
   ```

## Notes

- `coursesetter` and `play` are deprecated apps and are **excluded** from
  extraction (see `EXCLUDE_DIRS` in `scripts/manage_translations.py`). The live surface is
  whatever the root `urls.py` routes to.
- `templates/guide.html` (+ `static/js/guide.js`, `static/css/guide.css`) is **not
  yet wired into `urls.py`**, so it was left untranslated. Translate it when the
  `/guide/` route is added.
- The Django admin localizes itself via Django's bundled translations; the custom
  `templates/admin/*` strings resolve through the same merged catalog.
- IT translations were produced without a native reviewer ŌĆö flag for review.
